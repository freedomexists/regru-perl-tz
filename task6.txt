Браузер клиента связывается с DNS-сервером (если данные не обнаружились в кеше), получает соответствующий введенному пользователем
URL IP-адрес и информацию по какому протоколу работает целевой сервер, HTTP или HTTPS. 
Теперь у браузера есть необходимая информация для установки соединения с сервером. Эта информация отдается на транспортный уровень,
где в заголовок помещается порт назначения (80й для HTTP, 443й для HTTPS), затем отдается на сетевой уровень,
где появляется понятный для этого уровня заголовок с IP адресом назначения и отправления. 
Сформированный пакет отдается на канальный уровень, где пакет получает заголовок с адресами понятными для этого уровня – МАС адресом машины назначения и текущей машины.
Наконец все готово для физической отправки сигнала содержащего эту информацию по линиям связи провайдеру и далее по узлам сети пока пакет не попадет на искомый адрес.
Сервер обрабатывает полученный пакет и отправляет запрос на установление соединения клиенту. 
Учитывая, что рег.ру использует HTTPS, далее сервер и клиент договариваются о методах и протоколах шифрования. Идет сложный процесс генерации и обмена ключами шифрования.
Затем сервер в ответ на зашифрованный GET-запрос, передает зашифрованный пакет, который помимо множества заголовков содержит в теле HTML и CSS код (возможно даже JS),
который наш браузер рендерит и отображает на экране.
